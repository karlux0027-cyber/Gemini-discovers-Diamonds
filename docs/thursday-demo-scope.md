# Thursday demo scope — “show how we create it”

**Audience:** Keith and family already know what a finished YouTube video looks like. The demo is **the creation pipeline**, not a generic “we posted a clip.”

**Definition of “live”:** A real **published video on YouTube** (public or unlisted). That counts as live—not necessarily YouTube Live streaming.

**Deadline:** Have the **real product** far enough along by **Thursday** so you can walk them through it in person while they’re in Palm Springs. Manual editing remains an acceptable **fallback** if automation is blocked; the goal is still to **ship the full system** (weekend push continues the same spine).

## One-sentence demo promise

**Upload a clip → see job status move through the pipeline → get a processed output → one explicit step publishes to YouTube (unlisted is fine for the first run).**

## What to show (minimum credible demo)

1. **Ingest** — One clear way to get footage in (upload UI, folder drop, or shared drive → bucket).
2. **State** — Visible job states (e.g. received → processing → ready for review → publishing → published / failed).
3. **Process** — At least one automated step they can see (e.g. normalize/transcode, or generate a proxy, or burn in a simple title slate—whatever is real in your build by Thursday).
4. **Publish** — Final action that sends **one** asset to YouTube so they can open it on a phone.

## Explicitly out of scope for Thursday (unless already free)

- Perfect multi-platform (TikTok/Reels) in one click.
- AI that “edits like a human” end-to-end.
- Polished B-roll logic—stub or rule-of-one is enough to show where it plugs in later.

## After Thursday

Keep building the **full pipeline** (review gate, shorts from master, B-roll library, richer automation) using the same job spine so nothing thrown together for the demo becomes a dead end.
